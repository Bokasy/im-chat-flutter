# 基于 Flutter 的即时通信 APP 综合设计报告

---

## 一、背景

随着移动互联网的快速发展，即时通信已经成为人们日常生活中不可或缺的沟通方式。微信、QQ、Telegram 等即时通信应用深刻改变了人们的社交和工作模式。本项目以课程综合设计为契机，基于 Flutter 跨平台框架与 Spring Boot 后端框架，设计并实现了一款功能完整、界面美观的即时通信移动应用。

本项目采用前后端分离架构，前端使用 Flutter 实现 Android/iOS/Web 三端统一的用户界面，后端使用 Spring Boot 提供 RESTful API 和 WebSocket 实时通信服务。数据库采用 MySQL 8.0 存储持久化数据，Redis 用于缓存和离线消息队列。项目实现了用户注册登录、好友管理、私聊群聊、消息撤回转发回复、扫码登录、消息搜索等核心功能，具备完整的即时通信应用能力。

**项目技术栈：**

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端框架 | Flutter 3.x | 跨平台移动应用开发框架 |
| 状态管理 | GetX | 状态管理、依赖注入、路由管理 |
| 网络请求 | Dio | HTTP 客户端 |
| 实时通信 | WebSocket | 基于 STOMP 的实时消息推送 |
| 后端框架 | Spring Boot 3.2 | 核心框架 |
| 持久层 | MyBatis-Plus | ORM 框架 |
| 数据库 | MySQL 8.0 | 关系型数据库 |
| 缓存 | Redis | 缓存 + 消息队列 |
| 认证 | JWT | Token 鉴权机制 |
| API 文档 | Swagger/SpringDoc | 接口文档 |

#### 3. 核心组件设计

**消息气泡组件（MessageBubble）**
- 根据 msgType 区分文本消息和图片消息
- 支持已读/未读状态标识（双勾图标）
- 支持引用消息摘要展示
- 撤回消息显示提示文字
- 长按触发操作菜单（BottomSheet）

**头像组件（AvatarWidget）**
- 使用 Image.network 加载网络头像
- 支持加载中占位符和加载失败回退
- 支持自定义边框颜色和宽度

**响应式布局（ResponsiveWrapper）**
- 宽屏（>800px）居中显示420px宽度，模拟手机效果
- 窄屏（≤800px）全屏显示
- 左右填充深色背景，与主题统一

#### 4. 状态管理

采用 GetX 框架进行状态管理：

- **AuthController**：管理用户认证状态，处理登录/注册/退出
- **ChatController**：管理聊天会话和消息，处理消息发送/接收/撤回/转发
- **ContactController**：管理好友列表和群组列表，处理好友申请

#### 5. 网络通信

- **REST API**：通过 Dio 发起 HTTP 请求，自动携带 JWT Token
- **WebSocket**：建立长连接实现实时消息推送，支持心跳保活和断线重连
- **消息类型**：CHAT（聊天）、ACK（确认）、TYPING（输入状态）、HEARTBEAT（心跳）

#### 3. WebSocket 实时通信

WebSocket 连接地址：`ws://服务器地址:8080/ws/chat?token=JWT_TOKEN`

**消息协议格式：**

```json
{
  "type": "CHAT|ACK|TYPING|HEARTBEAT",
  "receiverId": 123,
  "chatType": 1,
  "msgType": 1,
  "content": "消息内容",
  "mediaUrl": "图片URL",
  "replyMsgId": "引用消息ID",
  "atUserIds": "@用户ID列表"
}
```

**消息类型说明：**
- `CHAT`：聊天消息（文本/图片）
- `ACK`：消息确认回执
- `TYPING`：对方正在输入状态
- `HEARTBEAT`：心跳保活（每30秒）

**群聊消息广播逻辑：**
当 chatType=2 时，后端查询群成员列表，遍历所有在线成员逐一推送消息，离线成员的消息存入 Redis 队列。

#### 4. 认证与安全

- **JWT Token 认证**：登录成功后生成 Token，有效期7天
- **拦截器校验**：除登录/注册/扫码接口外，所有 API 需携带有效 Token
- **密码存储**：数据库存储用户密码（课程演示使用明文，生产环境应使用 BCrypt）
- **WebSocket 认证**：握手阶段从 URL 参数提取 Token 进行校验

#### 5. 统一响应格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {},
  "timestamp": 1717000000000
}
```

- 输入框采用深色填充样式，无边框设计

## 四、总结

本项目基于 Flutter + Spring Boot 技术栈，实现了一款功能完整的即时通信应用。项目采用前后端分离架构，前端使用 GetX 进行状态管理，后端提供 RESTful API 和 WebSocket 实时通信服务。数据库设计合理，支持私聊、群聊、好友管理等核心业务。

项目的主要技术亮点包括：
1. 基于 WebSocket 的实时消息推送，支持私聊和群聊
2. JWT Token 身份认证机制，支持扫码登录
3. 消息可靠性保障：唯一ID、序列号、ACK确认、离线存储
4. GetX 响应式状态管理，UI 实时更新
5. 跨平台一套代码，Android/iOS/Web 三端运行
6. 深色主题 UI 设计，统一的视觉风格
