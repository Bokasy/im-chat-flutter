# 项目需求规格说明书

## 1. 项目概述

### 1.1 项目名称
基于 Flutter 的即时通信 APP 开发

### 1.2 项目背景
本项目为 Flutter 移动应用开发课程的综合作业，要求小组（≤5人）完成一个具备即时通信功能的移动应用，支持 Web/手机端部署演示。

### 1.3 项目目标
- 实现基于 WebSocket 的实时即时通信功能
- 完成用户鉴权、好友管理、聊天记录等核心业务
- 支持 Android/iOS/Web 多端部署运行
- 实现多端扫码登录、数据同步等亮点功能
- UI 界面美观，交互流畅

## 3. 功能需求

### 3.1 用户模块

#### 3.1.1 用户注册
- **功能描述**：新用户注册账号
- **输入项**：用户名、密码、昵称、头像（可选）
- **业务规则**：
  - 用户名唯一性校验
  - 密码强度要求（≥6位，包含字母和数字）
  - 注册成功后自动登录

#### 3.1.2 用户登录
- **功能描述**：用户身份认证
- **输入项**：用户名、密码
- **业务规则**：
  - 支持 Token 认证（JWT）
  - 登录状态缓存至 Redis
  - 登录过期时间：7天
  - 支持多端同时登录（可配置单端/多端策略）

#### 3.1.3 多端扫码登录 ⭐
- **功能描述**：Web 端通过手机端扫码登录
- **流程说明**：
  1. Web 端生成登录二维码（含唯一 token）
  2. 二维码有效期：5分钟
  3. 手机端扫码后确认登录
  4. Web 端自动登录，同步聊天记录
- **业务规则**：
  - 二维码定时刷新（30秒轮询）
  - 扫码后手机端显示登录设备信息
  - 支持踢出其他设备

#### 3.1.4 用户信息管理
- **功能描述**：查看/编辑个人资料
- **可编辑项**：昵称、头像、个性签名、兴趣标签、在线状态
- **业务规则**：
  - 头像支持上传（自动压缩）+ OSS 存储
  - 兴趣标签支持多选
  - 在线状态：在线/离线/忙碌/勿扰

### 3.2 好友模块

#### 3.2.1 好友搜索
- **功能描述**：根据用户名/昵称搜索用户
- **输入项**：搜索关键词
- **输出项**：用户列表（头像、昵称、签名、在线状态）

#### 3.2.2 好友申请
- **功能描述**：发送好友申请
- **业务规则**：
  - 不能重复申请
  - 不能添加已为好友的用户
  - 支持申请消息附言

#### 3.2.3 好友申请审核 ⭐
- **功能描述**：处理收到的好友申请
- **操作选项**：
  - **接受**：建立好友关系，通知对方
  - **拒绝**：拒绝申请，可选填拒绝理由
  - **忽略**：申请标记为已读，稍后处理
- **业务规则**：
  - 申请消息实时推送
  - 支持批量操作
  - 申请记录保留30天

#### 3.2.4 好友管理
- **功能描述**：好友列表展示、好友信息查看
- **功能项**：
  - 好友列表（按首字母分组）
  - 好友详情查看
  - 好友删除（双向删除）
  - 好友备注设置
  - 好友分组管理（可选）

### 3.3 聊天模块

#### 3.3.1 私聊
- **功能描述**：一对一实时聊天
- **消息类型**：
  - 文本消息
  - 图片消息（压缩 + OSS 存储）
  - 语音消息（可选）
  - 文件消息（可选）
- **业务规则**：
  - 基于 WebSocket (STOMP) 实时推送
  - 消息持久化存储至 MySQL
  - 支持消息已读/未读状态
  - 消息发送失败自动重试（最多3次）

#### 3.3.2 群聊
- **功能描述**：多人实时聊天
- **群组功能**：
  - 创建群组
  - 邀请/移除成员
  - 群公告
  - 群昵称设置
- **业务规则**：
  - 群消息广播给所有在线成员
  - 离线消息存储，上线后推送
  - 群成员上限：500人

#### 3.3.3 消息撤回 ⭐
- **功能描述**：撤回已发送的消息
- **业务规则**：
  - 撤回时限：2分钟内
  - 撤回后显示「你撤回了一条消息」
  - 对方显示「对方撤回了一条消息」
  - 群聊中仅自己和管理员可撤回

#### 3.3.4 消息转发 ⭐
- **功能描述**：将消息转发给其他好友/群组
- **业务规则**：
  - 支持单条/多条消息转发
  - 支持逐条转发/合并转发
  - 转发消息显示来源

#### 3.3.5 @某人功能 ⭐
- **功能描述**：在群聊中@指定成员
- **业务规则**：
  - 输入@触发成员列表
  - 被@成员收到特殊提醒
  - 消息中@高亮显示
  - 支持@所有人（仅管理员）

#### 3.3.6 消息引用/回复 ⭐
- **功能描述**：引用某条消息进行回复
- **业务规则**：
  - 长按消息选择回复
  - 回复显示原消息摘要
  - 点击原消息可跳转定位

#### 3.3.7 输入状态提示 ⭐
- **功能描述**：显示对方正在输入...
- **业务规则**：
  - 实时推送输入状态
  - 显示「对方正在输入...」
  - 5秒无输入自动消失

#### 3.3.8 消息已读状态 ⭐
- **功能描述**：显示消息已读/未读
- **业务规则**：
  - 私聊：已读/未读双状态
  - 群聊：显示已读人数
  - 已读状态实时更新

### 3.4 会话模块

#### 3.4.1 会话列表
- **功能描述**：展示所有聊天会话
- **展示信息**：头像、昵称、最新消息、时间戳、未读数
- **业务规则**：
  - 按最新消息时间倒序排列
  - 未读消息数显示
  - 支持置顶/删除会话
  - 支持消息免打扰 ⭐
  - 支持会话归档 ⭐

#### 3.4.2 消息免打扰 ⭐
- **功能描述**：设置会话免打扰
- **业务规则**：
  - 免打扰会话不显示未读数
  - 免打扰会话不触发通知提醒
  - 支持全局免打扰设置

#### 3.4.3 会话归档 ⭐
- **功能描述**：归档不常用的会话
- **业务规则**：
  - 归档会话移入单独分组
  - 新消息自动取消归档
  - 支持批量归档

#### 3.4.4 消息搜索 ⭐
- **功能描述**：搜索聊天记录
- **搜索范围**：
  - 全局搜索
  - 会话内搜索
- **搜索条件**：关键词、时间范围、消息类型

### 3.5 离线消息与多端同步

#### 3.5.1 离线消息推送 ⭐
- **功能描述**：用户离线时的消息处理
- **业务规则**：
  - 离线期间消息存储至 Redis 队列
  - 上线后批量拉取离线消息
  - 支持离线消息数量提醒
  - 可选：推送通知（需集成 FCM/APNs）

#### 3.5.2 多端数据同步 ⭐
- **功能描述**：多设备间数据同步
- **同步内容**：
  - 聊天记录
  - 会话状态（置顶、免打扰）
  - 好友列表
  - 未读状态
- **业务规则**：
  - 采用增量同步策略
  - 最后登录设备为主设备
  - 支持设备管理

### 3.6 消息安全

#### 3.6.1 消息可靠性保障
- **消息有序性**：基于序列号保证消息顺序
- **消息重发机制**：发送失败自动重试（指数退避）
- **消息确认机制**：客户端 ACK 确认接收
- **消息幂等性**：基于消息 ID 去重

#### 3.6.2 阅后即焚（可选）⭐
- **功能描述**：消息阅读后自动销毁
- **业务规则**：
  - 发送方可设置焚毁时间（5秒/30秒/1分钟）
  - 阅读后倒计时显示
  - 倒计时结束消息消失

## 5. 接口设计规范

### 5.1 RESTful API 规范
- 基础路径：`/api/v1`
- 请求方式：GET/POST/PUT/DELETE
- 响应格式：JSON
- 认证方式：JWT Token（Header: Authorization）

### 5.2 统一响应格式
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {},
  "timestamp": 1717000000000
}
```

### 5.3 分页响应格式
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "records": [],
    "total": 100,
    "size": 20,
    "current": 1
  }
}
```

## 6. 非功能需求

### 6.1 性能要求
- 消息发送延迟 < 500ms
- 页面加载时间 < 2s
- 支持 1000+ 并发连接
- 消息吞吐量 > 1000条/秒

### 6.2 安全要求
- 密码加密存储（BCrypt）
- JWT Token 认证（支持刷新）
- SQL 注入防护（MyBatis-Plus 参数化）
- XSS 防护
- 接口限流（Sentinel）
- 敏感数据脱敏

### 6.3 可靠性要求
- WebSocket 断线自动重连（指数退避）
- 消息发送失败自动重试（最多3次）
- 消息有序性保证（序列号）
- 消息幂等性（消息ID去重）

### 6.4 兼容性要求
- Android 5.0+
- iOS 11.0+
- Web 端（Chrome/Edge/Safari）
