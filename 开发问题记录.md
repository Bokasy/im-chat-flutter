# 开发问题记录

---

## 1. Flutter SDK 损坏导致编译失败
- **时间**：项目初期
- **问题**：Flutter SDK 版本号异常（3.35.5），`Offset`、`Rect`、`Color` 等基础类型无法识别，所有文件编译报错
- **原因**：下载的 Flutter SDK 为非官方或损坏版本
- **解决**：删除旧 SDK，从 Flutter 官网下载稳定版（3.24.5），重新配置环境变量

---

## 2. dio 与 get 包类型冲突
- **时间**：项目中期
- **问题**：`FormData` 和 `MultipartFile` 在 `dio` 和 `get` 两个包中都有定义，导致 `ambiguous_import` 编译错误
- **原因**：两个包导出了同名类
- **解决**：在 import get 时隐藏冲突类型：`import 'package:get/get.dart' hide Response, FormData, MultipartFile;`

---

## 3. 头像图片不显示（SVG 格式）
- **时间**：Web 端调试阶段
- **问题**：dicebear 头像 URL 返回 SVG 格式，Flutter Web 的 `CachedNetworkImage` 不支持 SVG 渲染，头像显示为空白
- **原因**：数据库中头像 URL 使用 `/svg?` 格式
- **解决**：将数据库中所有头像 URL 从 `/svg?` 改为 `/png?`，并将 `AvatarWidget` 从 `CachedNetworkImage` 改为 `Image.network`

---

## 4. withValues() API 兼容性问题
- **时间**：Web 端调试阶段
- **问题**：`Color.withValues(alpha: 0.1)` 是 Flutter 3.27+ 新 API，低版本不支持导致编译报错
- **原因**：项目中有 6 个文件使用了此新 API
- **解决**：全局替换为 `Color.withOpacity(0.1)`

---

## 5. WebSocket 连接后无法发送消息
- **时间**：双人聊天测试阶段
- **问题**：两个标签页登录不同账号后互相发消息，消息发不出去
- **原因**：`_isConnected` 标志只在收到消息时才设为 `true`，但 `sendMessage()` 发送前会检查此标志，导致连接建立后第一条消息被拦截
- **解决**：在 `connect()` 方法中 WebSocket 连接建立后立即将 `_isConnected` 设为 `true`

---

## 6. 多标签页 token 互相覆盖
- **时间**：双人聊天测试阶段
- **问题**：浏览器开两个标签页分别登录不同账号，第二个标签页登录后覆盖了第一个的 token，导致第一个标签页的 WebSocket 断开
- **原因**：`shared_preferences` 在 Web 端使用 `localStorage`，同一域名下所有标签页共享
- **解决**：Web 端改用内存存储（`Map`），每个标签页独立，互不干扰

---

## 7. 登录后会话列表为空
- **时间**：双人聊天测试阶段
- **问题**：登录后聊天列表为空，需要手动刷新才能看到会话
- **原因**：`ChatController` 的 `loadSessions()` 在 `onInit()` 时调用，此时用户还未登录，API 调用必然失败
- **解决**：在 `AuthController.login()` 和 `register()` 成功后主动调用 `ChatController.loadSessions()`

---

## 8. 扫码登录页面 Timer 内存泄漏
- **时间**：自检阶段
- **问题**：二维码倒计时 `Timer.periodic` 未存储引用，`dispose()` 时无法取消，导致页面关闭后 Timer 继续运行
- **原因**：匿名 Timer 没有赋值给变量
- **解决**：新增 `_countdownTimer` 字段存储引用，`dispose()` 中同时取消 `_pollTimer` 和 `_countdownTimer`

---

## 9. 后端扫码登录 userId 为 null
- **时间**：自检阶段
- **问题**：`POST /api/v1/user/login/qrcode/scan` 接口的 `request.getAttribute("userId")` 始终返回 null
- **原因**：`WebConfig` 中 JWT 拦截器排除了 `/api/v1/user/login/qrcode/**`，导致扫码确认接口也不经过 JWT 校验
- **解决**：将排除路径改为只排除 `generate` 和 `check`，保留 `scan` 接口的 JWT 校验

---

## 10. 后端 WebSocket 消息字段缺失导致 NPE
- **时间**：自检阶段
- **问题**：客户端发送的消息缺少 `receiverId` 或 `chatType` 字段时，后端直接 `null.toString()` 抛出空指针异常
- **原因**：`handleChatMessage` 方法未做参数校验
- **解决**：在解析前添加 null 检查，字段缺失时打印警告并 return

---

## 11. 文件上传大小配置不一致
- **时间**：自检阶段
- **问题**：后端 `FileServiceImpl` 允许 50MB，但 Spring 配置 `max-file-size` 只有 10MB，大文件被 Spring 拦截而非业务层处理
- **原因**：两处配置值不同
- **解决**：将 `application.yml` 中 `max-file-size` 和 `max-request-size` 统一改为 50MB

---

## 12. 前端缺少 WebSocketService 导入
- **时间**：扫码登录功能开发后
- **问题**：`qrcode_login_page.dart` 中使用 `Get.find<WebSocketService>()` 但未导入对应包，编译报 `non_type_as_type_argument`
- **原因**：新增代码时遗漏 import
- **解决**：添加 `import 'package:im_flutter/services/websocket_service.dart';`

---

## 13. 编辑资料页面未真正调用 API
- **时间**：自检阶段
- **问题**：点击保存后显示"已保存"，但实际只有 TODO 注释，未调用后端接口
- **原因**：`_saveProfile()` 方法未实现
- **解决**：注入 `ApiService`，调用 `PUT /user/update` 接口，成功后更新本地存储并刷新用户信息

---

## 14. 后端 POST 接口 @RequestParam 与前端请求不匹配
- **时间**：双人聊天测试阶段
- **问题**：添加好友、撤回消息、标记已读等接口报 `MissingServletRequestParameterException: Required request parameter 'targetId' not present`
- **原因**：后端用 `@RequestParam`（URL 查询参数）接收，但前端用 `data: {...}` 发送 POST body。Spring 无法从 body 中解析 `@RequestParam`
- **解决**：将所有 POST 接口的 `@RequestParam` 改为 `@RequestBody Map<String, Object>`，从 body 中手动取值

---

## 15. 后端 Integer 比较用 == 导致潜在 bug
- **时间**：自检阶段
- **问题**：`chatType == 1` 等比较使用 `==` 对 Integer 对象，超出 -128~127 范围会失败
- **原因**：Java Integer 缓存机制，`==` 比较的是引用而非值
- **解决**：改为 `Integer.valueOf(1).equals(chatType)`

## 16. WebSocket 断开时误删重连的新 session
- **时间**：自检阶段
- **问题**：用户重连后旧 session 的 `afterConnectionClosed` 回调把新 session 从 map 中移除
- **原因**：`SESSION_MAP.remove(userId)` 无条件删除
- **解决**：改为 `SESSION_MAP.remove(userId, session)` 条件删除

## 17. 前端 auth_controller 空指针崩溃
- **时间**：自检阶段
- **问题**：`data['userInfo']['id']` 未做 null 检查，服务器返回异常数据时崩溃
- **原因**：直接链式访问未防御
- **解决**：添加 `if (data == null || data['userInfo'] == null)` 检查

## 18. 前端 setState 在 await 后未检查 mounted
- **时间**：自检阶段
- **问题**：API 异步调用返回后 widget 已销毁，调用 setState 报错
- **原因**：await 后直接 setState，未判断 mounted 状态
- **解决**：添加 `if (mounted)` 守卫

## 19. 多标签页 localStorage 共享导致 token 覆盖
- **时间**：双人聊天测试阶段
- **问题**：浏览器开两个标签页分别登录不同账号，第二个登录覆盖第一个的 token
- **原因**：`shared_preferences` 在 Web 端使用 `localStorage`，同域名所有标签页共享
- **解决**：Web 端改用内存存储（`_WebStorage` 类），每个标签页独立

---

## 16. WebSocket 连接后消息发不出去
- **时间**：双人聊天测试阶段
- **问题**：WebSocket 连接建立后发送消息失败
- **原因**：`_isConnected` 标志只在收到消息时设为 `true`，连接建立时未设置
- **解决**：在 `connect()` 方法中 WebSocket 连接建立后立即将 `_isConnected` 设为 `true`

---

## 17. 前端请求日志过多干扰调试
- **时间**：Web端调试阶段
- **问题**：ApiService 拦截器打印每个请求的 URL 和响应状态码，控制台输出过多
- **原因**：拦截器中 `print('请求: ...')` 和 `print('响应: ...')` 在每次请求时都会执行
- **解决**：移除请求和响应的 print 语句，仅保留错误日志

---

## 18. ListTile 背景色遮挡水波纹效果
- **时间**：Web端调试阶段
- **问题**：Flutter 报警告 `ListTile background color or ink splashes may be invisible`
- **原因**：ListTile 外层 Container 设置了背景色，遮挡了 Material 水波纹效果
- **解决**：将 Container 的背景色移到 `Material` 组件上，ListTile 包裹在 Material 内

---

## 19. Get.arguments 空指针风险
- **时间**：自检阶段
- **问题**：`chat_detail_page`、`user_card_page` 等页面直接 `Get.arguments['key']`，若未传参数则崩溃
- **原因**：未做 null 安全处理
- **解决**：改为 `final args = Get.arguments as Map<String, dynamic>? ?? {};` 后再取值
